--------------------------------------------------------------------------------
  ЩО ЦЕ ТАКЕ
--------------------------------------------------------------------------------
Веб-сайт для відстеження приросту Galactic Power (GP) гравців у грі
Star Wars: Galaxy of Heroes (SWGOH).

Сайт показує прогрес 6 гільдій (~50 гравців кожна) та 6 особистих друзів.
Для друзів також доступна детальна сторінка з переліком змін у ростері
(які персонажі/кораблі прокачав за день).

Сайт закритий — потребує логіну.

--------------------------------------------------------------------------------
  ТЕХНІЧНИЙ СТЕК
--------------------------------------------------------------------------------
  Backend:     Python 3.11 + FastAPI
  База даних:  SQLite (файл /data/gp_tracker.db на Railway Volume)
  Frontend:    HTML + Alpine.js + Chart.js
  Хостинг:     Railway (безкоштовний план)
  API даних:   swgoh-comlink (окремий сервіс на Railway)
               URL: https://swgoh-comlink-production-26de.up.railway.app
  Автозбір:    cron-job.org (три завдання на день)

--------------------------------------------------------------------------------
  БАЗА ДАНИХ — ТАБЛИЦІ
--------------------------------------------------------------------------------

  snapshots
  — Основна таблиця GP. Один запис на гравця на день.
  — Поля: snapshot_date, guild_id, player_id, player_name, gp, is_final
  — is_final=1 тільки для знімку о 00:00 (використовується для стріку)
  — UNIQUE(snapshot_date, guild_id, player_id) — перезаписується при повторному зборі

  settings
  — Налаштування сайту. Зараз: monthly_plan (місячний план GP).
  — Поля: key, value

  roster_snapshots
  — Стан ростеру кожного друга. Один запис на юніт на день.
  — Поля: snapshot_date, player_id, unit_id, current_level, gear_tier,
           relic_tier, current_stars, combat_type
  — Зберігається тільки для друзів (не для гільдій)

  roster_ability_snapshots
  — Рівні скілів кожного юніта кожного друга. Один запис на скіл на день.
  — Поля: snapshot_date, player_id, unit_id, ability_id, tier, is_zeta

  unit_names
  — Кеш назв юнітів та іконок. Завантажується один раз кнопкою "Reload".
  — Поля: unit_id, name, combat_type, thumbnail_name
  — Джерело: GitHub swgoh-utils/gamedata + CDN game-assets.swgoh.gg

--------------------------------------------------------------------------------
  ЗБІР ДАНИХ — ЯК ПРАЦЮЄ
--------------------------------------------------------------------------------
  Три рази на день cron-job.org відправляє POST /api/cron:

  1. 06:00 Київ — ранковий (проміжний, is_final=0)
  2. 15:00 Київ — денний (проміжний, is_final=0)
  3. 00:00 Київ — фінальний (is_final=1) → URL: /api/cron?final=1

  При кожному зборі:
  — Для 6 гільдій: запит до comlink → зберігаємо GP кожного гравця
  — Для 6 друзів: запит до comlink → зберігаємо GP + ростер (юніти + скіли)

  Дані дня перезаписуються (INSERT OR REPLACE). Зберігається завжди остання
  версія за поточний день.

  Лімітів comlink: не більше 20 запитів/сек. Між запитами є затримка 0.2-0.3с.

--------------------------------------------------------------------------------
  ГОЛОВНА СТОРІНКА (index.html)
--------------------------------------------------------------------------------
  Вкладки:
  — Friends     — таблиця 6 друзів
  — 6 гільдій   — таблиця ~50 гравців кожна

  Колонки таблиці:
  — #           — позиція + зміна рангу (↑↓ на скільки місць змінився рейтинг
                  по GP Growth порівняно з попереднім снапшотом)
                  Оновлюється після кожного з трьох кронів на день.
  — 7 days      — 7 міні-стовпчиків: зелений=GP виріс, червоний=не виріс, темний=немає даних
  — Player      — нікнейм (для друзів — клікабельний, веде на сторінку гравця)
  — GP Now      — поточний GP
  — GP Growth   — приріст з 1-го числа місяця до сьогодні
                  (тільки гільдії) поруч сірим — зміна з попереднього крону (+/-)
  — Streak      — дні підряд з позитивним/негативним приростом
  — Plan X%     — відсоток виконання місячного плану (план в налаштуваннях ⚙)

  Фільтр місяця — перемикач для перегляду даних за попередні місяці.
  Сортування    — по будь-якій колонці (клік на заголовок).

  Ачівки (тільки Friends, перші 7 днів місяця, за результати минулого місяця):
  — 🥇🥈🥉  — топ-3 за приростом GP
  — 🐌        — найменший приріст
  — ❄️        — нульовий приріст (не грав)
  — ☠️        — від'ємний GP

  Streak ачівки (в колонці Streak):
  — 🔥        — +7 днів підряд позитивний стрік
  — 🥶        — -3 і нижче підряд негативний стрік

  Стрік рахується тільки по is_final=1 знімках (00:00).
  Якщо фінальних знімків менше 2 — fallback на всі знімки (для старих даних),
  але сьогоднішній незавершений знімок (is_final=0) завжди виключається —
  щоб ранковий крон (06:00) не скидав стрік до гравців, які ще не грали.

  Плашка NEW (тільки для гільдій):
  — З'являється поруч з іменем гравця якщо він у гільдії ≤ 7 днів
  — Формат: [NEW] joined DD.MM
  — Визначається за першою датою появи у знімках цієї гільдії
  — Довгі імена обрізаються з ellipsis, повне ім'я — у title (hover)

  Кнопка ⚙ — налаштування (єдина кнопка в хедері):
    — Місячний план GP
    — Unit names: статус завантаження + кнопка Reload
    — Download backup — завантаження бекапу бази даних
    — Refresh all data — ручний збір даних (потребує COLLECT_PASSWORD)

  "Updated: YYYY-MM-DD HH:MM" — час останнього збору по Києву (Europe/Kyiv),
    відображається в хедері Friends і кожної гільдії. Зберігається в settings.

--------------------------------------------------------------------------------
  СТОРІНКА ГРАВЦЯ (/player/details?id=...&name=...)
--------------------------------------------------------------------------------
  Доступна тільки для друзів. Відкривається кліком на нікнейм у вкладці Friends.

  Ім'я гравця — по центру вгорі, великим шрифтом.
  Під іменем — місячні ачівки (якщо є, перші 7 днів місяця).

  Stats-рядок (під іменем):
  — Rank     — позиція + зміна відносно вчора (↑↓)
  — GP Now   — поточний GP
  — GP Growth — приріст з 1-го числа місяця
  — Streak   — дні підряд + 🔥 якщо ≥ +7, 🥶 якщо ≤ -3
  — Plan X%  — відсоток виконання місячного плану
  — Activity — кольорова крапка + рівень 1-10

  Банер (тільки якщо назви юнітів не завантажено):
  — ⚠️ Unit names not loaded — перейти в ⚙ на головній для завантаження

  Блок "UPGRADED YESTERDAY":
  — Картки персонажів прокачаних вчора: апгрейди + нові юніти (NEW)
  — Кожна картка: іконка + назва + опис змін (R8→R9, G12→G13 тощо)
  — Якщо нічого не прокачував → "No upgrades yesterday"
  — Порівнює останній знімок вчорашнього дня з позавчорашнім

  Блок "GP History" видалено.

  Блок "[МІСЯЦЬ] UPGRADES" (під календарем):
  — Всі апгрейди за вибраний місяць одним списком
  — Кожен персонаж показується один раз з фінальним результатом
    (наприклад G8→G13, навіть якщо прокачувався поступово за кілька днів)
  — Порівнює перший і останній знімок місяця
  — Нові персонажі — з плашкою NEW
  — Оновлюється разом з календарем при зміні місяця

  Блок "MARCH 2026" (календар поточного місяця):
  — Сітка 7 колонок (Mon-Sun), один рядок = один тиждень
  — Всі рядки тижня однакової висоти (розтягуються під найвищу клітинку)
  — Клітинки:
      Зелена  — є знімок + є зміни (гравець щось прокачав)
      Червона — є знімок, але змін немає (грав, але нічого не апгрейдив)
      Сіра    — немає знімка (майбутній день або до початку збору)
  — У зелених клітинках — текст: назва персонажа + що змінилось
  — Якщо змін більше 4 — показується "+N more"
  — Вибір місяця: клік на заголовок "MARCH 2026 ▾" → відкривається пікер
      Пікер: сітка 3×4 місяців + кнопки ← рік → для навігації по роках
      Недоступні місяці (до початку збору або майбутні) — сірі

  Кнопки "Download report in TXT" — завантаження звіту за місяць або рік.
  — Файл: gp_YYYY-MM.txt (місяць) або gp_YYYY.txt (рік)
  — Формат: англійська, UTF-8, plaintext
  — Завантажується за поточним місяцем календаря (змінюється через піккер ▾)

  Для кожного юніта в змінах показується:
  — Іконка персонажа (CDN: game-assets.swgoh.gg/textures/{thumbnailName}.png)
  — Назва + тип (Character / Ship)
  — Кількість зірок (7⭐) — для кораблів тільки зірки
  — Gear рівень з кольором (тільки для персонажів без релікa):
      G1           — сірий
      G2-G4        — світло-зелений
      G5-G7        — світло-синій
      G8-G11       — фіолетовий
      G12 / G13    — золотий
  — Relic рівень (R1-R9+), формула: relic_tier_stored - 2
      (stored 3=R1, stored 11=R9, stored 12=R10)
  — Зміни скілів (рівень ability підвищився)

  Якщо GP виріс, але жодних змін у ростері/скілах немає → гравець апгрейдив моди.

--------------------------------------------------------------------------------
  API ENDPOINT'И
--------------------------------------------------------------------------------
  GET  /                              — головна сторінка
  GET  /login                         — сторінка логіну
  POST /login                         — авторизація (встановлює cookie)
  GET  /player/details                — сторінка гравця (query: id, name)

  GET  /api/guilds                    — список гільдій
  GET  /api/progress/{guild_id}       — прогрес гільдії (query: month)
  GET  /api/months/{guild_id}         — доступні місяці для гільдії
  GET  /api/friends/history           — GP-графік для друзів (Chart.js)
  GET  /api/friends/achievements      — ачівки для друзів
  GET  /api/friends/list              — список друзів з player_id
  GET  /api/settings                  — поточний monthly_plan
  POST /api/settings                  — оновлення monthly_plan
  GET  /api/status                    — статус поточного збору (для прогрес-бару)
  GET  /api/backup                    — завантаження бекапу БД
  POST /api/collect                   — ручний збір (потребує COLLECT_PASSWORD)
  POST /api/cron                      — автоматичний збір (потребує X-Cron-Token)
                                        query: final=1 для фінального знімку

  GET  /api/friends/roster_changes/{player_id}  — зміни ростеру (query: date)
  GET  /api/friends/roster_dates/{player_id}    — доступні дати знімків ростеру
  GET  /api/friends/roster_month/{player_id}    — зміни за місяць (query: month=YYYY-MM)
  GET  /api/friends/roster_month_summary/{player_id} — підсумок місяця: перший vs останній знімок
  GET  /api/friends/export/{player_id}          — завантажити TXT звіт
                                                  query: report_type=month&month=YYYY-MM
                                                      або report_type=year&year=YYYY

  POST /api/admin/sync_unit_names     — завантажити/оновити назви юнітів
  GET  /api/admin/unit_names_status   — кількість збережених назв юнітів
  DELETE /api/admin/roster_snapshot/{date} — видалити знімки ростеру за дату

--------------------------------------------------------------------------------
  ВІДОМІ ОБМЕЖЕННЯ / НОТАТКИ
--------------------------------------------------------------------------------
  — GP окремого юніта недоступний через comlink API (тільки сумарний GP гравця)
  — Моди не відстежуються прямо. Якщо GP ріс, але змін у ростері немає → моди.
  — Localization bundle service для comlink не розгорнутий — назви юнітів
    завантажуються з GitHub swgoh-utils/gamedata.
  — Рівень relic у comlink: stored_value - 2 = реальний R-рівень
    (stored 11 = R9, stored 12 = R10, stored 3 = R1 тощо)
  — Датакрони не впливають на GP і не відстежуються.
  — Ростер та сторінка гравця — тільки для друзів (не для гільдій).
